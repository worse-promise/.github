# Worse Promise/S+.js
Worse Promise/Schrödinger+ is a powerful promise implementation for enterprise environments where unreliability is needed over reliability.

## Features
- 0 reliability
- 10x engineering
- 67 rizz
- free v-bucks
- 0.1% a+ spec compliance at least
- 0 tests
- async
- promise-like
- duck typed

This is called a Schrödinger+ promise, because it is both success and failure at the same time, but also even AFTER when it is observed (the "+").
